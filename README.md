## Purpose of this repository
This repository is created for SWE 442 Homework 1 to demonstrate the software configuration management skills including (branching, pull requests, and issue tracking).

## Best Practices in Software Configuration Management
1. **Version Control Everything** – Code, docs, configs, scripts.
2. **Use Meaningful Commit Messages** – Explain what and why.
3. **Branch Strategically** – Use feature branches and pull requests.
4. **Tag Releases** – Mark stable versions.
5. **Automate Testing** – Run tests on every push/PR.
6. **Review Code** – Never merge without review.
